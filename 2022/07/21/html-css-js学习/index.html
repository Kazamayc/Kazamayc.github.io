0x01 url和file协议和http协议

url：统一资源定位符，用来表示从互联网上得到的资源位置和访问这些资源的方法，互联网上的所有资源，都有一个确定的url。由于访问不同对象所使用的协议不同，所以url还要指出读取某个对象时所使用的协议。

协议（protocol）：通信计算机双方必须共同遵守的一组约定，包括怎样建立连接，最终体现为在网络上传输的数据包的格式。最终体现为网络上传输的数据包的格式。

file协议：本地文件传输协议，主要用于访问本地计算机中的文件。只有一个类似http的远程访问，

file协议///的原因是因为只能在本地传输，所以没有主机，中间的部分省略了。

http协议：超文本传输协议，是用来从万维网服务器传输超文本到本地浏览器的传输协议，基于TCP/IP通信协议来传输数据。

http协议工作在客户端-服务端的架构上，由浏览器作为http客户端通过url向http服务端即web服务器（例如apache、ngnix、iis）发送请求，web服务器对其进行解析后，知道了要调度哪些资源文件，再通过请求所需的调用数据库信息，最后向客户端发送响应信息。
